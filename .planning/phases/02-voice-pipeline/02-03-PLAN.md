**Context:** CONTEXT.md locked decision states "Streaming TTS response where possible to reduce perceived latency." RESEARCH.md Pitfall 4 warns that sequential pipeline (STT 1s + Claude 1.5s + TTS 1s = 3.5s+) will exceed 4 seconds. The fix: sentence-boundary streaming — accumulate Claude's streaming text until a sentence boundary (. ! ?) is detected, then dispatch each sentence to TTS immediately while Claude continues generating. This reduces perceived latency to ~1.5-1.6s (STT 1s + Claude first sentence 300ms + TTS first sentence 75-300ms).

**1. Create `app/voice/streaming.py`** — Sentence-boundary streaming module:

Implement `async def stream_conversation_to_audio(user_text: str, language: str = "tr", history: list[dict] | None = None, detected_language: str | None = None) -> bytes`:

This function replaces the sequential `run_conversation() -> tts_service.synthesize()` pipeline with a streaming one:

Step 1 — Stream Claude response with sentence accumulation:
```python
import anthropic
import asyncio
import re
from app.config import settings
from app.voice.conversation import VELAR_SYSTEM_PROMPT
from app.voice.tts import tts_service

SENTENCE_BOUNDARY_RE = re.compile(r'(?<=[.!?])\s+')

async def stream_conversation_to_audio(
    user_text: str,
    language: str = "tr",
    history: list[dict] | None = None,
    detected_language: str | None = None,
) -> tuple[str, bytes]:
    """
    Stream Claude response through sentence-boundary TTS.
    Returns (full_response_text, concatenated_audio_bytes).

    Strategy: Accumulate Claude's streamed text. When a sentence boundary
    (. ! ?) followed by whitespace is detected, dispatch that sentence to
    TTS immediately via asyncio.create_task while Claude continues generating.
    After Claude finishes, dispatch any remaining text, then await all TTS
    tasks and concatenate audio in order.
    """
```

Step 2 — Build the system prompt (same logic as conversation.py, with language context):
```python
    system = VELAR_SYSTEM_PROMPT
    if detected_language:
        lang_name = {"tr": "Turkish", "en": "English"}.get(detected_language, detected_language)
        system += f"\n\n[Context: The user is speaking {lang_name}. Respond in {lang_name}.]"

    messages = ((history or [])[-10:]) + [{"role": "user", "content": user_text}]
```

Step 3 — Stream from Claude using `client.messages.stream()` (the anthropic SDK supports streaming context manager). Accumulate text, detect sentence boundaries, dispatch TTS tasks:
```python
    client = anthropic.Anthropic(api_key=settings.anthropic_api_key)

    buffer = ""           # Accumulates streamed text
    full_text = ""        # Complete response text
    tts_tasks = []        # Ordered list of asyncio tasks producing audio bytes

    # Use asyncio.to_thread for the streaming iteration since SDK is sync
    def _stream_claude():
        """Generator yielding text deltas from Claude streaming response."""
        with client.messages.stream(
            model="claude-haiku-4-5-20251001",
            max_tokens=512,
            system=system,
            messages=messages,
        ) as stream:
            for text in stream.text_stream:
                yield text

    # Collect all deltas via to_thread (stream is sync)
    deltas = await asyncio.to_thread(lambda: list(_stream_claude()))

    for delta in deltas:
        buffer += delta
        full_text += delta

        # Check for sentence boundary: text ending with . ! ? followed by space or at end
        while True:
            match = SENTENCE_BOUNDARY_RE.search(buffer)
            if not match:
                break
            # Split at the boundary
            sentence = buffer[:match.start() + 1].strip()  # Include the punctuation
            buffer = buffer[match.end():]

            if sentence:
                # Dispatch this sentence to TTS immediately
                task = asyncio.create_task(
                    tts_service.synthesize(text=sentence, language=language)
                )
                tts_tasks.append(task)

    # Dispatch any remaining text in buffer (last sentence may not end with boundary)
    remaining = buffer.strip()
    if remaining:
        task = asyncio.create_task(
            tts_service.synthesize(text=remaining, language=language)
        )
        tts_tasks.append(task)

    # Await all TTS tasks and concatenate audio in order
    audio_chunks = await asyncio.gather(*tts_tasks)
    combined_audio = b"".join(audio_chunks)

    return full_text, combined_audio
```

**Important design notes:**
- The function returns `(full_text, audio_bytes)` so the caller has both the response text (for metadata headers) and the concatenated audio.
- TTS tasks run concurrently with each other. The first sentence starts TTS while later sentences are still being accumulated. This is where the latency win comes from.
- `asyncio.gather(*tts_tasks)` preserves order, so audio chunks are concatenated in sentence order.
- The streaming SDK uses `client.messages.stream()` context manager which yields text deltas. We collect them via `to_thread` since the SDK is synchronous.
- If Claude returns a very short response (1 sentence), this degrades gracefully to the same behavior as the non-streaming path.
