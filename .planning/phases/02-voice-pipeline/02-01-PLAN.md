```python
from pydantic import BaseModel, Field

class STTResult(BaseModel):
    text: str
    language: str                      # "tr" or "en"
    language_probability: float        # 0.0-1.0

class ChatRequest(BaseModel):
    message: str = Field(..., min_length=1, max_length=4000)

class ChatResponse(BaseModel):
    text: str
    audio_base64: str                  # MP3 audio as base64
    detected_language: str

class VoiceMetadata(BaseModel):
    transcript: str
    response_text: str
    detected_language: str
```

**5. Create `app/voice/stt.py`** — the STT service:

Key implementation details:
- Load WhisperModel as a module-level singleton (NOT per-request). Constructor takes model_size from settings.whisper_model_size.
- Use `device="cpu"`, `compute_type="int8"` (Mac/Windows compatible, no CUDA dependency).
- `async def transcribe(self, audio_bytes: bytes) -> STTResult` — main entry point.
- Internally uses `asyncio.to_thread(self._transcribe_sync, audio_bytes)` to avoid blocking.
- `_transcribe_sync` method:
  1. Uses a `threading.Lock` around the model to prevent concurrent access corruption.
  2. Tries to decode audio with soundfile first (handles WAV, FLAC, OGG).
  3. If soundfile fails, falls back to pydub: `AudioSegment.from_file(io.BytesIO(audio_bytes)).export(format="wav")` then re-reads with soundfile. This handles WebM, M4A, MP4 from browsers/phones.
  4. Calls `self.model.transcribe(audio_array, vad_filter=True, vad_parameters=dict(min_silence_duration_ms=500), beam_size=5)` — do NOT set `language=` parameter. Let Whisper auto-detect.
  5. Joins segment texts with spaces. Returns STTResult with text, language, language_probability.
  6. If `info.language_probability < 0.8`, log a warning (useful for debugging short utterance detection issues per Research pitfall 6).
- Do NOT create the stt_service singleton at module level. Instead, create a `get_stt_service()` function that lazily initializes the singleton on first call. This way, importing the module doesn't trigger the 5-20 second model download/load at import time — it only happens when actually needed (at app startup via lifespan or first request). This is critical for tests that import the module but don't need the actual model.

**Important anti-patterns to avoid:**
- Do NOT force `language="tr"` — this breaks English input and code-switching detection.
- Do NOT load WhisperModel per request (5-20s init time).
- Do NOT call model.transcribe() from async code without asyncio.to_thread.

**Test 4: `test_audio_format_conversion`** — Unit test. Create a simple WAV in memory using soundfile (sine wave, 1 second, 16kHz). Pass to `_transcribe_sync` with a mock WhisperModel, verifying that the audio decoding path works:

```python
async def test_audio_format_conversion():
    """WAV bytes are correctly decoded to numpy float32 array."""
    import numpy as np
    import soundfile as sf
    import io
    # Generate 1 second of silence at 16kHz
    audio = np.zeros(16000, dtype=np.float32)
    buf = io.BytesIO()
    sf.write(buf, audio, 16000, format="WAV")
    wav_bytes = buf.getvalue()
    # Verify soundfile can read it back
    buf2 = io.BytesIO(wav_bytes)
    data, sr = sf.read(buf2, dtype="float32")
    assert sr == 16000
    assert len(data) == 16000
```
